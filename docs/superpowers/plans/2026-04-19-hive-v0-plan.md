# Hive v0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement Hive v0 — a 14-region distributed LLM brain where each region is a containerized Python process communicating exclusively over MQTT — per the approved spec at [docs/superpowers/specs/2026-04-19-hive-v0-design.md](docs/superpowers/specs/2026-04-19-hive-v0-design.md).

**Architecture:** Single shared `region_template/` package (DNA, P-VI) baked into one `hive-region:v0` Docker image; 14 containerized regions differentiated only by `HIVE_REGION` env var + bind-mounted `regions/<name>/` directory (gene expression). A `glia/` supervisor container manages lifecycle (launch, heartbeat monitor, rollback, ACL rendering) via the Docker socket. A single `mosquitto` broker enforces topic ACLs. All inter-region communication is JSON-envelope over MQTT.

**Tech Stack:** Python 3.11 · `aiomqtt` 2.x (asyncio MQTT) · LiteLLM 1.x (LLM abstraction) · mosquitto 2.0.x (broker) · Docker + Compose v2 · `structlog` 24.x (JSON logs) · `ruamel.yaml` (comment-preserving YAML) · `jsonschema` 4.x (Draft 2020-12) · `testcontainers` (component tests) · `pytest` + `pytest-asyncio`.

**Spec reference convention:** Tasks cite spec sections (e.g., "§A.3") rather than duplicating full code. Implementer MUST read the referenced section before writing code.

## How to use this plan

1. Work tasks in order within a phase. Phases themselves are ordered; don't skip ahead.
2. Every code task follows TDD: write failing test, verify failure, implement, verify pass, commit. Configuration-only tasks skip tests.
3. Use `@spec §X.Y` to know where to read the canonical signatures/schemas. The spec is authoritative; the plan is the schedule.
4. Commit after every task (granular history is required by P-XII). Commit messages follow the pattern `<phase>: <concise summary>` e.g., `runtime: add config_loader + tests`.
5. Parallel-dispatch tasks are flagged **PARALLEL-OK**. Any phase without that flag is sequential.
6. When a task surfaces a spec gap or ambiguity, STOP and surface it before guessing. Spec changes go through the open-questions loop; do not silently override.
